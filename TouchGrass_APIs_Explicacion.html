01 / Frontend

HTML5, CSS3 y JavaScript

La base de toda la aplicación. Sin frameworks, sin compiladores. Solo el navegador.

🌐
Vanilla JavaScript ES2022 + HTML5 + CSS3 Sin frameworks
Toda la lógica de la aplicación está escrita en JavaScript puro, sin React, Vue ni Angular. Esto permite cargar la app directamente desde GitHub Pages sin ningún proceso de build.

Por qué este enfoque

Una aplicación móvil-first que se sirve como página web estática necesita el menor overhead posible. Vanilla JS arranca en milisegundos y no requiere Node.js, npm ni Webpack. Los archivos se sirven tal cual están en el repositorio de GitHub.

Estructura de archivos

  • index.html — Toda la estructura HTML de la app (pantallas, modales, navegación). Una sola página (SPA manual)
  • css/styles.css — Sistema de diseño completo: variables CSS, tipografía, componentes, dark mode nativo
  • css/context.css — Estilos de los componentes dinámicos: banner de clima, botones de encuesta, energía
  • css/tracker.css — Panel del tracker GPS, placeholder, badges de grupo, Street View
  • js/app.js — Lógica principal: autenticación, grupos, leaderboard, chat, navegación
  • js/challenges.js — Motor de retos: selección, tracker GPS, renderizado, completar
  • js/context.js — Detección de contexto: clima, ciudad, tipo automático de reto
  • js/maps.js — Integración con Google Maps: Street View, geocodificación, mapa estático
  • js/firebase-config.js — Inicialización de Firebase con las credenciales del proyecto

Patrón SPA (Single Page Application)

El HTML define varias "pantallas" (.screen) y "páginas" (.page) que se muestran u ocultan añadiendo/quitando la clase active. Nunca hay recarga de página. La navegación es instantánea.

// app.js — Cambiar pantalla activa function showScreen(id) { document.querySelectorAll('.screen').forEach(s => s.classList.remove('active')); document.getElementById(id)?.classList.add('active'); } // Navegar a una sección sin recargar la página function navigateTo(pageId) { document.querySelectorAll('.page').forEach(p => p.classList.remove('active')); document.getElementById(`page-${pageId}`)?.classList.add('active'); }

Variables CSS como sistema de diseño

Todo el sistema visual (colores, tipografía, espaciados, radios) está definido en variables CSS en :root. Cambiar un color en una variable lo actualiza en toda la aplicación automáticamente.

/* styles.css */ :root { --bg-primary: #0d0d0f; --accent: #1fce36; /* verde principal */ --font-display: 'Syne', sans-serif; /* títulos */ --font-body: 'Space Grotesk', sans-serif; /* cuerpo */ --radius-lg: 20px; /* bordes redondeados */ }
02 / Autenticación

Firebase Auth — Google OAuth

El sistema que identifica a cada usuario de forma segura, sin contraseñas.

🔐
Firebase Authentication FIREBASE SDK
Permite a los usuarios iniciar sesión con su cuenta de Google con un solo clic. Firebase gestiona tokens, sesiones y seguridad automáticamente.

Qué es Firebase Auth

Firebase Authentication es un servicio de Google que actúa como intermediario entre la app y el proveedor de identidad (en este caso, Google). Cuando el usuario pulsa "Continuar con Google", Firebase abre un popup seguro de Google, el usuario autoriza la app, y Firebase devuelve un objeto user con su UID único, nombre, email y foto. La app nunca maneja contraseñas.

Cómo se carga en la app

<!-- index.html: SDK de Firebase cargado antes de todo --> <script src="https://www.gstatic.com/firebasejs/10.12.0/firebase-app-compat.js"></script> <script src="https://www.gstatic.com/firebasejs/10.12.0/firebase-auth-compat.js"></script> // firebase-config.js: inicializar con las credenciales del proyecto firebase.initializeApp(firebaseConfig); const auth = firebase.auth(); const googleProvider = new firebase.auth.GoogleAuthProvider();

Flujo de autenticación

1
Usuario pulsa "Continuar con Google" Se llama a auth.signInWithPopup(googleProvider). Firebase abre el popup de Google de forma segura.
2
Google autentica al usuario Google verifica credenciales y devuelve un token OAuth 2.0 a Firebase. El usuario ve la pantalla de selección de cuenta de Google.
3
Firebase notifica a la app auth.onAuthStateChanged(user => {...}) se dispara. El objeto user contiene UID, nombre, email y foto.
4
La app crea o carga el perfil Se consulta Firestore: si el UID existe, se carga el perfil. Si no existe, se crea un documento nuevo con XP=0, racha=0, grupos vacíos.

Código en app.js

// Escuchar cambios de sesión (se ejecuta al cargar y al login/logout) auth.onAuthStateChanged(async (user) => { if (!user) { showScreen('screen-auth'); // sin sesión → pantalla login return; } // Con sesión: cargar datos del usuario de Firestore await loadOrCreateUser(user); showScreen('screen-app'); // mostrar app principal CTX_init(); // detectar clima y ubicación initChallenges(); // cargar reto del día });
Seguridad Firebase Auth garantiza que el UID de cada usuario es único e infalsificable. Las reglas de seguridad de Firestore pueden verificar ese UID para que cada usuario solo acceda a sus propios datos.
03 / Base de datos

Firebase Firestore

Base de datos NoSQL en tiempo real que sincroniza datos entre todos los usuarios del grupo automáticamente.

🔥
Cloud Firestore NOSQL · TIEMPO REAL
Almacena perfiles de usuario, grupos, mensajes de chat, notificaciones y el estado de retos. Los cambios se sincronizan en todos los dispositivos conectados en tiempo real sin necesidad de recargar.

Estructura de la base de datos

Firestore (NoSQL — colecciones de documentos) │ ├── users/{uid} │ ├── name, email, photoURL │ ├── xp, coins, streak │ ├── groupIds: [id1, id2, ...] ← multi-grupo │ └── completedChallenges: [...] │ ├── groups/{groupId} │ ├── name, code, ownerId │ └── members: [uid1, uid2, ...] │ ├── messages/{msgId} │ ├── groupId, uid, name │ ├── text, timestamp │ └── notifications/{notifId} ├── toUid, type, text └── day, unread, date

Listeners en tiempo real

Firestore permite suscribirse a un documento o colección. Cada vez que los datos cambian en el servidor, el callback se ejecuta automáticamente en todos los clientes suscritos. Esto permite que el leaderboard y el chat se actualicen en tiempo real sin polling.

// app.js — Leaderboard en tiempo real // onSnapshot dispara el callback cada vez que hay cambios db.collection('users') .where('groupIds', 'array-contains', groupId) .onSnapshot((snap) => { const members = snap.docs .map(doc => doc.data()) .sort((a, b) => b.xp - a.xp); // ordenar por XP renderLeaderboard('leaderboard-full', members); });

Operaciones usadas en la app

  • db.collection('users').doc(uid).get() — Leer perfil del usuario al hacer login
  • db.collection('users').doc(uid).set(data) — Crear perfil nuevo si es primera vez
  • db.collection('users').doc(uid).update(fields) — Actualizar XP, monedas, racha al completar un reto
  • db.collection('groups').add(data) — Crear un grupo nuevo con código único
  • db.collection('groups').where('code','==', code).get() — Buscar grupo por código para unirse
  • db.collection('messages').onSnapshot() — Escuchar mensajes del chat en tiempo real
  • FieldValue.arrayUnion(groupId) — Añadir un groupId al array sin sobreescribir los demás
04 / GPS del dispositivo

Web Geolocation API

API nativa del navegador que accede al GPS del dispositivo. No requiere ninguna librería externa.

📍
navigator.geolocation NATIVA · SIN KEY
API estándar del navegador W3C que solicita acceso al GPS del dispositivo. Funciona en móvil y escritorio. El usuario debe conceder permiso explícitamente. Usada en dos contextos distintos en la app.

Uso 1: Detectar la ciudad del usuario (una vez al cargar)

Al inicializar la app, se obtiene la posición una sola vez para saber en qué ciudad está el usuario. Estas coordenadas se envían a Open-Meteo (clima) y a Nominatim (nombre de ciudad).

// context.js — Obtener posición única function CTX_requestLocation() { return new Promise((resolve, reject) => { navigator.geolocation.getCurrentPosition( pos => resolve({ lat: pos.coords.latitude, lon: pos.coords.longitude }), err => reject(err), { timeout: 10000, maximumAge: 300000 } // cache 5 min ); }); }

Uso 2: Tracker GPS continuo (al correr/caminar)

Cuando el usuario activa el tracker para un reto exterior, se usa watchPosition en vez de getCurrentPosition. La diferencia es que watchPosition notifica cada vez que la posición cambia, permitiendo dibujar la ruta y calcular los kilómetros recorridos en tiempo real, igual que Strava.

// challenges.js — Tracker GPS tipo Strava _trackerWatchId = navigator.geolocation.watchPosition( (pos) => { const pt = { lat: pos.coords.latitude, lon: pos.coords.longitude, ts: Date.now() }; // Calcular distancia respecto al punto anterior (fórmula haversine) if (_trackerPoints.length > 0) { const delta = _haversineKm(prev.lat, prev.lon, pt.lat, pt.lon); _trackerKm += delta; // acumular km } _trackerPoints.push(pt); _trackerUpdateUI(); // actualizar contador en pantalla _trackerCheckArrival(pt.lat, pt.lon); // ¿llegó al checkpoint? }, (err) => console.warn(err), { enableHighAccuracy: true, timeout: 10000, maximumAge: 5000 } ); // Al parar el tracker, dejar de escuchar el GPS navigator.geolocation.clearWatch(_trackerWatchId);

Fórmula Haversine — Calcular distancia entre coordenadas GPS

Para calcular los kilómetros recorridos, la app usa la fórmula haversine, que calcula la distancia entre dos puntos sobre la superficie esférica de la Tierra a partir de sus coordenadas de latitud y longitud. Es la misma fórmula que usan apps como Strava o Google Maps para medir distancias.

// challenges.js — Distancia entre dos puntos GPS en km function _haversineKm(lat1, lon1, lat2, lon2) { const R = 6371; // radio de la Tierra en km const dLat = (lat2 - lat1) * Math.PI / 180; const dLon = (lon2 - lon1) * Math.PI / 180; const a = Math.sin(dLat/2)**2 + Math.cos(lat1 * Math.PI/180) * Math.cos(lat2 * Math.PI/180) * Math.sin(dLon/2)**2; return R * 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1-a)); }

Detección de llegada al checkpoint (80 metros)

Algunos retos requieren llegar a un punto concreto (ej: la Catedral de Santiago). El tracker comprueba en cada actualización de posición si la distancia al checkpoint es menor de 80 metros. Si es así, el reto se completa automáticamente.

// challenges.js — Comprobar llegada al checkpoint const distM = _haversineKm(lat, lon, cpLat, cpLon) * 1000; // metros if (distM <= 80) { // ARRIVAL_RADIUS_METERS = 80 TRACKER_stop(); showToast('🎯 ¡Has llegado al punto de control!', 'success'); }
Permiso requerido El navegador siempre pide permiso al usuario antes de acceder al GPS. Si el usuario deniega el permiso, la app funciona igualmente pero sin geolocalización: no detecta clima, no activa el tracker ni detecta la llegada al checkpoint.
05 / Clima

Open-Meteo API

Datos meteorológicos en tiempo real, completamente gratuitos y sin necesidad de API key.

🌤️
Open-Meteo (api.open-meteo.com) GRATIS · SIN KEY
API meteorológica de código abierto que proporciona temperatura actual y código de tiempo (WMO) para cualquier coordenada GPS del mundo. La app la usa para decidir automáticamente si proponer un reto interior o exterior según el tiempo.

Qué datos devuelve y para qué se usan

🌡️ temperature_2m

Temperatura actual en grados Celsius a 2 metros del suelo. Se muestra en el banner de clima: "Santiago · Nublado · 14°C"

☁️ weather_code (WMO)

Código numérico estándar WMO que describe el estado del cielo (0=despejado, 61=lluvia, 95=tormenta...). Decide si el reto es exterior o interior.

Cómo se construye la petición

// context.js — Petición a Open-Meteo async function CTX_fetchWeather(lat, lon) { const url = `https://api.open-meteo.com/v1/forecast` + `?latitude=${lat.toFixed(4)}&longitude=${lon.toFixed(4)}` + `¤t=temperature_2m,weather_code` // campos solicitados + `&timezone=auto`; // zona horaria automática const res = await fetch(url); const data = await res.json(); return { temp: Math.round(data.current.temperature_2m), code: data.current.weather_code // código WMO }; }

Tabla de códigos WMO implementada en la app

La app tiene una tabla de traducción de códigos WMO a descripción, emoji y si el tiempo es apto para exterior. Así decide automáticamente qué tipo de reto proponer:

// context.js — Tabla WMO const _WMO = { 0: { emoji: '☀️', desc: 'Despejado', outdoor: true }, 3: { emoji: '☁️', desc: 'Nublado', outdoor: true }, 61: { emoji: '🌧️', desc: 'Lluvia ligera', outdoor: false }, 95: { emoji: '⛈️', desc: 'Tormenta', outdoor: false }, // ... 15 códigos en total }; // Si outdoor=true → proponer reto exterior automáticamente window.CTX.autoType = info.outdoor ? 'outdoor' : 'indoor';

Efecto en la UI: el banner de clima

Tras obtener el clima, se inyecta dinámicamente un banner en la parte superior de la app con el emoji del tiempo, la descripción, la temperatura y la ciudad. El banner se auto-oculta tras 7 segundos con una animación CSS.

06 / Geocodificación inversa

Nominatim / OpenStreetMap

Convierte coordenadas GPS (latitud, longitud) en un nombre de ciudad legible. Sin coste, sin registro.

🗺️
Nominatim (nominatim.openstreetmap.org) GRATIS · SIN KEY
Servicio de geocodificación inversa de OpenStreetMap. Dado un par de coordenadas, devuelve la dirección y ciudad correspondientes. Se usa exclusivamente para mostrar el nombre de la ciudad en el banner de clima.

Qué es geocodificación inversa

La geocodificación directa convierte un nombre ("Madrid") en coordenadas. La geocodificación inversa hace el proceso contrario: convierte coordenadas (40.4168, -3.7038) en un nombre legible ("Madrid"). Nominatim usa la base de datos cartográfica de OpenStreetMap, que es colaborativa y de libre acceso.

Cómo se usa en el código

// context.js — Obtener nombre de ciudad async function CTX_fetchCity(lat, lon) { const url = `https://nominatim.openstreetmap.org/reverse` + `?lat=${lat}&lon=${lon}&format=json&accept-language=es`; const res = await fetch(url, { headers: { 'User-Agent': 'MoveUp-Hackathon/1.0' } // requerido por Nominatim }); const data = await res.json(); // Extraer el nombre más específico disponible return data.address?.city || data.address?.town || data.address?.village || data.address?.county || 'Tu ciudad'; // fallback si no hay datos }
Open-Meteo + Nominatim: en paralelo Ambas peticiones se lanzan simultáneamente con Promise.all([CTX_fetchWeather(), CTX_fetchCity()]). Esto reduce el tiempo de espera a la duración de la petición más lenta (no se suman).
07 / Google Maps

Google Maps JavaScript API

El núcleo de toda la integración con Google Maps. Una única clave de API que activa múltiples servicios.

🗺️
Maps JavaScript API REQUIERE API KEY GOOGLE CLOUD
SDK de JavaScript de Google Maps que se carga de forma asíncrona en el HTML. Una vez cargado, expone el objeto global google.maps con todos los servicios: mapas interactivos, Street View, geocodificación y Places.

Cómo se carga el SDK

El SDK se carga con un tag <script> al final del HTML, con el atributo async defer para no bloquear el renderizado de la página, y el parámetro callback=MAPS_onLoad que indica a Google qué función ejecutar cuando el SDK esté listo.

<!-- index.html — Al final del body, después de todos los demás scripts --> <script src="https://maps.googleapis.com/maps/api/js ?key=TU_API_KEY &callback=MAPS_onLoad &libraries=places &loading=async" <!-- requerido desde Maps JS v3.55 --> async defer> </script>

El callback MAPS_onLoad

Cuando Google termina de cargar el SDK, llama automáticamente a la función MAPS_onLoad definida en maps.js. Esta función marca el SDK como listo e inicializa el geocodificador. Si el modal se abre antes de que Maps cargue, se muestra un placeholder con un enlace directo a Google Maps.

// maps.js — Callback ejecutado por Google cuando el SDK carga function MAPS_onLoad() { _mapsReady = true; _geocoder = new google.maps.Geocoder(); // inicializar geocodificador console.log('✅ Google Maps SDK listo'); } window.MAPS_onLoad = MAPS_onLoad; // exponer globalmente para que Google lo encuentre

Por qué loading=async en la URL

Desde la versión 3.55 de Maps JS API (2024), Google exige incluir el parámetro loading=async en la URL del script para activar el modo de carga optimizado. Sin él, la API funciona pero lanza una advertencia en la consola y puede tener peor rendimiento. Es independiente del async defer del tag HTML.

Servicios incluidos en el SDK

Una vez cargado el SDK de Maps JS, quedan disponibles automáticamente varios servicios a través del objeto google.maps:

  • google.maps.Geocoder — Geocodificación directa e inversa
  • google.maps.StreetViewPanorama — Street View interactivo
  • google.maps.StreetViewService — Comprobar disponibilidad de Street View
  • google.maps.Map — Mapa interactivo (no usado directamente, pero disponible)
08 / Geocodificación

Google Geocoding API

Convierte el nombre de un lugar ("Catedral de Santiago de Compostela") en coordenadas GPS precisas.

📌
Geocoding API (vía google.maps.Geocoder) REQUIERE API KEY
Servicio de Google que traduce nombres de lugares, direcciones o puntos de interés en coordenadas geográficas exactas (latitud y longitud). Se usa para localizar los checkpoints de los retos en el mapa.

Para qué se usa en TouchGrass

Cada reto exterior con checkpoint tiene un campo checkpoint.query con el nombre del lugar en lenguaje natural. Cuando el usuario abre el modal de detalles del reto, la app geocodifica ese nombre para obtener las coordenadas exactas. Esas coordenadas se usan para tres cosas: mostrar Street View, ofrecer un link de "Cómo llegar" y detectar si el tracker llega al checkpoint.

Ejemplos de checkpoints en CHALLENGES_DB

// challenges.js — Algunos retos con checkpoint { id: 'treasure_hunt', checkpoint: { name: 'Catedral de Santiago', query: 'Catedral de Santiago de Compostela' } }, { id: 'run5k', checkpoint: { name: 'Parque de Bonaval', query: 'Parque de Bonaval Santiago de Compostela' } }, { id: 'walk_no_phone', checkpoint: { name: 'Alameda de Santiago', query: 'Alameda de Santiago de Compostela' } },

Cómo se realiza la geocodificación

// maps.js — Geocodificar el nombre del checkpoint _geocoder.geocode({ address: 'Catedral de Santiago de Compostela' }, (results, status) => { if (status !== 'OK' || !results.length) { _renderPlaceholder(container, query, 'No se encontró el lugar'); return; } // Extraer latitud y longitud del primer resultado const location = results[0].geometry.location; const lat = location.lat(); // ej: 42.8805 const lon = location.lng(); // ej: -8.5446 // Guardar en CTX para que el tracker las use window.CTX.checkpoint = { name: query, lat, lon }; // Comprobar si hay Street View disponible en esas coordenadas const svService = new google.maps.StreetViewService(); svService.getPanorama({ location: { lat, lng: lon }, radius: 100 }, ...); });
Cadena de datos Las coordenadas que devuelve la Geocoding API se guardan en window.CTX.checkpoint. El tracker GPS las lee en cada actualización de posición para calcular la distancia al objetivo y detectar la llegada automáticamente.
09 / Vista inmersiva

Google Street View

Muestra una vista panorámica interactiva del checkpoint del reto directamente en el modal de detalles.

👁️
StreetViewPanorama + StreetViewService REQUIERE API KEY INCLUIDO EN MAPS JS
Street View permite al usuario ver el lugar donde tiene que ir antes de salir. Se integra directamente en el modal de detalles del reto, convirtiendo la pantalla de información en una experiencia inmersiva e interactiva.

Por qué Street View mejora la experiencia

Ver el punto de destino antes de salir tiene un valor real: el usuario puede reconocer el lugar cuando llegue, orientarse mejor y sentir que el reto es concreto y alcanzable, no abstracto. El Street View es interactivo: el usuario puede girar la vista en 360°, moverse por la calle y acercar el zoom directamente desde la app.

Flujo de carga del Street View

1
Usuario abre "Ver detalles" del reto Se llama a MAPS_loadForCurrentChallenge() que comprueba si el reto tiene checkpoint y si es exterior.
2
Geocodificación del nombre del lugar _geocoder.geocode({ address: query }) convierte "Catedral de Santiago" en coordenadas GPS exactas.
3
Comprobar disponibilidad de Street View svService.getPanorama() verifica si hay imágenes de Street View en un radio de 100 metros alrededor del punto.
4a
Street View disponible → Panorama interactivo Se crea un StreetViewPanorama en el contenedor del modal. El usuario puede girar la vista.
4b
Street View no disponible → Mapa estático (fallback) Se llama a la Static Maps API para obtener una imagen satelital del lugar como alternativa.

Código de creación del panorama

// maps.js — Crear el panorama de Street View function _renderStreetViewPanorama(container, lat, lon, label) { container.innerHTML = ''; container.style.height = '220px'; const panoDiv = document.createElement('div'); panoDiv.style.width = panoDiv.style.height = '100%'; container.appendChild(panoDiv); // Crear el panorama interactivo de Street View _streetView = new google.maps.StreetViewPanorama(panoDiv, { position: { lat, lng: lon }, // coordenadas del checkpoint pov: { heading: 0, pitch: 0 }, // orientación inicial zoom: 1, addressControl: false, // ocultar dirección fullscreenControl: true, // botón pantalla completa motionTrackingControl: false, // no girar con el giroscopio showRoadLabels: true, // mostrar nombres de calles }); // Añadir link "Cómo llegar" debajo del panorama _appendMapsLink(container, lat, lon, label); }

Nota sobre "streetview" como librería

Street View no es una librería que se deba importar con libraries=streetview. Está incluido por defecto en el SDK de Maps JavaScript. Solo se necesita importar libraries=places (para la Places API). Intentar importar streetview provoca el error "The library streetview is unknown" en consola.

10 / Fallback visual

Google Maps Static API

Genera una imagen estática del mapa cuando Street View no está disponible en el checkpoint.

🖼️
Maps Static API REQUIERE API KEY FALLBACK
Devuelve una imagen PNG/JPEG de un mapa centrado en unas coordenadas concretas. No es interactivo: es una foto del mapa. Se usa como fallback cuando Street View no tiene cobertura en el punto del reto.

Cómo funciona: una URL, una imagen

La Static Maps API no requiere JavaScript. Es una URL con parámetros que devuelve directamente una imagen. Se puede usar en una etiqueta <img>. La API key se incluye en la URL como parámetro.

// maps.js — Construir la URL del mapa estático const staticUrl = `https://maps.googleapis.com/maps/api/staticmap` + `?center=${lat},${lon}` // centrar en el checkpoint + `&zoom=17` // nivel de zoom (17 = detalle de calle) + `&size=600x220` // dimensiones de la imagen en píxeles + `&maptype=roadmap` // tipo de mapa (roadmap/satellite/terrain) + `&markers=color:red%7C${lat},${lon}` // marcador rojo en el punto + `&key=${apiKey}`; // API key // Inyectar como imagen en el contenedor del modal container.innerHTML = `<img src="${staticUrl}" alt="${label}" .../>`;

Obtener la API key sin duplicarla en el código

Para no escribir la API key dos veces (ya está en el tag <script> de Maps JS), la app la extrae dinámicamente del atributo src de ese script mediante una regex:

// maps.js — Extraer la key del tag <script> de Google Maps function _getApiKey() { const scripts = document.querySelectorAll('script[src*="maps.googleapis.com"]'); for (const s of scripts) { const match = s.src.match(/[?&]key=([^&]+)/); if (match) return match[1]; } return null; }

Fallback sin API key: iframe de Google Maps

Si por alguna razón la key no está disponible, la app usa un <iframe> de Google Maps Embed, que permite mostrar un mapa básico sin key (con limitaciones de uso):

// maps.js — Fallback sin key: iframe embed container.innerHTML = ` <iframe src="https://maps.google.com/maps?q=${encodeURIComponent(label)}&output=embed" width="100%" height="220" style="border:0"> </iframe> `;
11 / Lugares

Google Places API

Librería de servicios de lugares incluida al cargar el SDK de Maps JS con libraries=places.

🏛️
Places API (libraries=places) REQUIERE API KEY ACTIVA EN CLOUD
Conjunto de servicios de Google para buscar, identificar y obtener detalles de lugares del mundo real: restaurantes, monumentos, tiendas, edificios públicos. En TouchGrass se carga como librería de apoyo a la geocodificación.

Por qué se activa en Google Cloud Console

Aunque el código actual usa principalmente el Geocoder estándar para los checkpoints, la librería places se importa porque proporciona capacidades adicionales de reconocimiento de lugares que el Geocoder básico no tiene: puede entender nombres ambiguos como "la Catedral" o "el Parque de Bonaval" con mayor precisión que el geocodificador genérico.

Cómo se importa

<!-- index.html — El parámetro &libraries=places activa el servicio --> <script src="https://maps.googleapis.com/maps/api/js ?key=AIzaSy... &libraries=places <!-- activa google.maps.places.* --> &callback=MAPS_onLoad &loading=async" async defer> </script> // Una vez cargado, disponible como: google.maps.places.PlacesService google.maps.places.Autocomplete google.maps.places.PlaceResult

Capacidades disponibles (para futuras versiones)

  • Autocompletado: Sugerir nombres de lugares mientras el usuario escribe, para crear retos con checkpoints personalizados
  • PlacesService: Buscar lugares por categoría (parques, monumentos) alrededor de la ubicación del usuario
  • Nearby Search: Encontrar automáticamente puntos de interés cercanos al usuario para generar checkpoints dinámicos y localizados
Habilitado en Google Cloud Places API debe estar habilitada en Google Cloud Console para que la importación funcione. Si no está habilitada, las llamadas a servicios de google.maps.places fallarán silenciosamente. Las APIs habilitadas en el proyecto son: Maps JavaScript API, Places API, Geocoding API, Maps Static API y Street View Static API.
12 / Visión global

Flujo completo de APIs encadenadas

Cómo se comunican todas las piezas desde que el usuario abre la app hasta que completa un reto exterior.

1
Login con Google Firebase Auth autentica al usuario con OAuth. Firestore carga o crea el perfil del usuario (XP, racha, grupos).
2
Detección de contexto (paralela) Web Geolocation API obtiene las coordenadas GPS. Simultáneamente: Open-Meteo devuelve temperatura y código de tiempo, y Nominatim devuelve el nombre de la ciudad.
3
Decisión automática de tipo de reto Si el código WMO indica buen tiempo → autoType = 'outdoor'. Si hay lluvia/niebla → autoType = 'indoor'. Se marca automáticamente el botón correspondiente en la encuesta.
4
El usuario elige energía → se asigna el reto La función pickChallengeForUser(type, energy, userId, date) usa un hash reproducible para asignar siempre el mismo reto al mismo usuario en el mismo día.
5
Usuario abre "Ver detalles" (reto exterior) Geocoding API de Google convierte el nombre del checkpoint en coordenadas. Se guardan en window.CTX.checkpoint.
6
Mostrar Street View o mapa estático StreetViewService comprueba disponibilidad. Si hay imágenes → StreetViewPanorama interactivo. Si no → Static Maps API como imagen de fallback.
7
Usuario activa el tracker GPS y sale a la calle Web Geolocation API en modo watchPosition actualiza la posición continuamente. La fórmula haversine acumula km. El SVG dibuja la ruta en tiempo real.
8
Llegada al checkpoint → Reto completado Cuando la distancia GPS al checkpoint es <80 metros, el reto se completa automáticamente. Firestore actualiza XP, monedas y racha. El leaderboard se actualiza en tiempo real para todos los miembros del grupo.
13 / Configuración

Google Cloud Console

Cómo están configuradas las APIs de Google en el proyecto de Google Cloud.

APIs habilitadas en el proyecto

🗺️ Maps JavaScript API

Carga el SDK principal de Google Maps. Sin esta API, nada de lo relacionado con mapas funciona. Es la base de la que dependen todas las demás.

📌 Geocoding API

Necesaria para que google.maps.Geocoder pueda convertir nombres de checkpoints en coordenadas GPS. Sin ella, el geocodificador devuelve error REQUEST_DENIED.

👁️ Street View Static API

Necesaria para que StreetViewService.getPanorama() pueda comprobar disponibilidad y para que el panorama interactivo funcione.

🖼️ Maps Static API

Necesaria para el fallback de imagen de mapa estático cuando Street View no tiene cobertura en el checkpoint.

🏛️ Places API

Importada como librería (libraries=places). Mejora la precisión de reconocimiento de lugares y habilita capacidades futuras de búsqueda de puntos de interés cercanos.

🔑 API Key única

Una sola API key con restricciones de dominio (localhost + dominio de producción) da acceso a todos los servicios de Google Maps listados arriba.

Restricciones de la API key

Para evitar uso no autorizado de la key, en Google Cloud Console se configuran restricciones en la API key:

  • Restricción de aplicación: Solo se permite el uso desde los dominios específicos de la app (localhost para desarrollo, dominio GitHub Pages para producción)
  • Restricción de API: La key solo puede llamar a las 5 APIs de la tabla anterior, no a otros servicios de Google

APIs de Google que NO requieren configuración

// Estas APIs son gratuitas y no requieren API key ni configuración en Cloud Open-Meteo → api.open-meteo.com // clima, completamente libre Nominatim → nominatim.openstreetmap.org // geocodificación inversa OSM Web Geolocation → navigator.geolocation // GPS nativo del navegador (W3C) Firebase → Plan Spark (gratuito) // auth + firestore Google Fonts → fonts.googleapis.com // tipografías canvas-confetti → CDN jsDelivr // animación de celebración

Para un hackathon o proyecto académico el uso es prácticamente siempre gratuito. La facturación solo se activa si se supera ese crédito, que requeriría decenas de miles de usuarios activos al mes.
